# Руководство по публикации на GitHub Pages

## Текущее состояние

Проект уже находится в git репозитории на ветке `gh-pages`, что является правильной настройкой для GitHub Pages.

## Шаги для публикации

### 1. Проверить настройки удаленного репозитория

```bash
git remote -v
```

Если удаленный репозиторий не настроен, нужно добавить его:

```bash
git remote set-url origin https://github.com/axtrace/doctor-queue-sim.github.io.git

https://github.com/axtrace/doctor-queue-sim.github.io.git
```

### 2. Добавить все новые файлы в git

```bash
git add .
```

### 3. Создать коммит с изменениями

```bash
git commit -m "Добавлена симуляция очереди к врачам

- Создана логика симуляции QueueSimulation.js
- Добавлен интерфейс управления QueueUI.js
- Интегрированы навигационные слайды
- Добавлены текстовые ресурсы на русском
- Созданы графические ассеты для симуляции"
```

### 4. Отправить изменения на GitHub

```bash
git push origin gh-pages
```

### 5. Настроить GitHub Pages (если нужно)

После отправки кода на GitHub:
1. Перейдите в настройки репозитория на GitHub
2. В разделе "Pages" убедитесь, что выбрана ветка `gh-pages`
3. Сохраните настройки

### 6. URL вашего сайта

Сайт будет доступен по адресу:
```
https://ВАШ_ЛОГИН.github.io/trust-queue-simulation/
```

## Структура проекта для GitHub Pages

Проект готов к публикации. Все необходимые файлы созданы:

### Основные файлы
- `index.html` - главная страница с интегрированной симуляцией
- `words.html` - текстовые ресурсы на русском языке
- `js/sims/QueueSimulation.js` - логика симуляции
- `js/sims/QueueUI.js` - интерфейс управления
- `js/slides/X_Slides_Queue.js` - навигационные слайды

### Ассеты
- `assets/queue/` - графические ресурсы для симуляции

### Тестовые файлы (можно удалить перед публикацией)
- `test_queue.html`, `test_main.html`, `test_navigation.html` - для локального тестирования
- `check_errors.js` - скрипт проверки ошибок
- `queue_simulation_report.md` - отчет о разработке

## Рекомендации по публикации

### Перед публикацией рекомендуется:
1. Удалить тестовые файлы:
```bash
rm test_queue.html test_main.html test_navigation.html check_errors.js
```

2. Обновить README.md с описанием новой симуляции

3. Проверить работу симуляции локально через `http://localhost:8000/`

### После публикации:
1. Проверить доступность сайта по URL GitHub Pages
2. Протестировать симуляцию в разных браузерах
3. При необходимости настроить кастомный домен (опционально)

## Особенности GitHub Pages

- Автоматическая сборка не требуется - проект статический
- Файлы обслуживаются напрямую из ветки `gh-pages`
- HTTPS включен по умолчанию
- Кэширование может занять несколько минут после публикации

## Альтернативный вариант: Fork оригинального репозитория

Если вы хотите сохранить связь с оригинальным проектом:

1. Перейдите на https://github.com/ncase/trust
2. Нажмите "Fork"
3. Клонируйте свой форк
4. Добавьте изменения с симуляцией очереди
5. Отправьте изменения в свой форк

Это позволит легко получать обновления от оригинального проекта.
